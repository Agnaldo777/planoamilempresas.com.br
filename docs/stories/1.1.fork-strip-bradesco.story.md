## Strategic Context / Why

Esta é a **primeira story de código** e a mais crítica do ponto de vista de higiene. O clone `amil-saude/` contém 1.005 páginas programáticas para Bradesco Saúde Empresarial. **Se qualquer referência "Bradesco" sobreviver ao strip**, o risco é duplo: (1) penalização Google por conteúdo incoerente (Amil site com referências Bradesco) e (2) risco legal (concorrente cita como cópia). O grep final `bradesco` = 0 matches em `src/`, `public/`, `docs/` é o AC mais crítico de toda esta story.

Ao mesmo tempo, **jogar fora os patterns é desperdício** — rate limiting Upstash, validação zod, route groups App Router, componentes Radix, SEO audit script são ativos valiosos que aceleram Epic 1-6 em 4-8 semanas.

O equilíbrio: **strip cirúrgico que elimina conteúdo mas preserva estrutura**.

## Risks & Mitigations

| Risk | Probabilidade | Impacto | Mitigation |
|------|:-------------:|:-------:|------------|
| Bradesco string sobrevive ao strip (em image alt, SVG, comment, data file) | M | H | grep recursivo multi-pass (case-insensitive) + manual review de `public/` assets (nomes de arquivo + binários com strings embeddadas) |
| Build quebra após strip (imports para dados removidos) | H | M | Criar stubs/placeholders para todas importações; testar build iterativamente durante strip, não só no final |
| Testes falham massivamente | M | L | Ajustar testes que referenciavam conteúdo específico; manter framework funcional; coverage pode cair temporariamente |
| `.env.local` do Bradesco commitada acidentalmente | L | H | Verificar `.gitignore` INCLUI `.env.local` + grep por API keys/tokens no diff final |
| Git history contém strings Bradesco (commits antigos) | H | L | Aceitável — `.git/` não é servido; audit trail preservado intencionalmente. Grep exclui `.git/` |
