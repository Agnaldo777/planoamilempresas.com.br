## Story

**As a** @qa (Quinn) e mantenedor do CI/CD,
**I want** GitHub Action obrigatória em cada PR que executa `scripts/validate-sitemap-routing.mjs` validando bidirectional consistency entre `app/sitemap.ts` (Next.js App Router) e routes descobertas em `app/**/page.tsx` (incluindo dynamic segments resolvidos por `generateStaticParams`),
**so that** o satélite cumpra NFR26 do PRD v1.3.1, evite o anti-padrão `amilsaudebr.com.br` (sitemap lista `/sao-paulo` mas links internos apontam para `/tabela-amil-sao-paulo-2026.html` — 2 conjuntos paralelos de URLs gerando confusão de indexação Google), e garanta integridade SEO em CI.

---

## Strategic Context / Why

`amilsaudebr.com.br` (research `09-amilsaudebr-kitcorretor-deep-scrape.md`) tem **sitemap inconsistente com links internos** — 14 URLs estaduais no sitemap (`/sao-paulo`, `/rio-de-janeiro`...) mas links internos da homepage apontam para padrão diferente (`/tabela-amil-sao-paulo-2026.html`). Google trata como dois conjuntos de URLs paralelas — penaliza com confusão de indexação ou indexação parcial.

Sem CI gate, o satélite (com 1.500+ URLs programmatic) tem alta probabilidade de regressão similar — esquecer de incluir nova rota no sitemap, ou listar URL no sitemap que foi renomeada em refactor. NFR26 torna esta validação **obrigatória em todo PR**, eliminando o erro classificado de "drift silencioso".

Story complementa Story 1.8 (Robots.txt + Sitemap.xml + SEO Foundation) — esta cobre **enforcement contínuo** do que aquela criou. ADR-008 confirma App Router; `app/sitemap.ts` é o pattern oficial.

## Acceptance Criteria

1. **Script `scripts/validate-sitemap-routing.mjs`**:
   - Parseia output de `app/sitemap.ts` executando `next build` + lendo `.next/server/app-paths-manifest.json` OU rodando `app/sitemap.ts` em isolated runner
   - Coleta lista A: todas URLs declaradas em sitemap
   - Discovery rotas Next.js via `glob` em `app/**/page.tsx` (incluindo route groups `(marketing)/`, dynamic segments `[slug]`, catch-all `[...slug]`)
   - Para dynamic segments: invoca `generateStaticParams` se exportado e expande URLs concretas
   - Coleta lista B: todas URLs roteáveis indexáveis (excluindo `app/api/*`, `noindex`-flagged pages)
   - Computa `setDifference(A, B)` e `setDifference(B, A)`
   - **Falha build se qualquer set não-vazio** (URL sitemap sem rota OU rota indexável sem sitemap entry)

2. **Tratamento de `noindex` pages:**
   - Pages com `metadata.robots.index = false` ou `<meta name="robots" content="noindex">` excluídas da lista B (não precisam estar no sitemap)
   - Páginas com `noindex` runtime (computed) detectadas via parsing AST ou run-time SSG simulation
   - Lista de exclusões intencionais documentada em `scripts/sitemap-routing.config.json` (ex: `/sandbox/`, `/api/`) <!-- RESOLVIDO 2026-05-01 (Pax): drift fix — rota `/studio` removida do `excludeFromSitemap` pois Sanity Studio agora é hosted em planoamilempresas.sanity.studio (Story 3.27). Validado com `node scripts/validate-sitemap-routing.mjs` Consistent. -->

3. **Tratamento de dynamic segments:**
   - `generateStaticParams` é executado dentro do CI (Next.js build) e URLs concretas comparadas
   - Catch-all routes (`[...slug]`) com generateStaticParams expandido
   - Story compatível com paginação routes (`/blog/page/[N]`) — sitemap deve incluir todas pages

4. **GitHub Action `.github/workflows/validate-sitemap-routing.yml`**:
   - Trigger: `pull_request` (todos PRs) + `push` para `main`
   - Steps: checkout → setup Node → `pnpm install` → `pnpm next build` → `node scripts/validate-sitemap-routing.mjs`
   - **Bloqueia merge** via branch protection rule "Required status check"
   - Comment automático no PR mostrando URLs em divergência (formato Markdown table)
   - Cache `.next/` para acelerar runs subsequentes

5. **Output friendly em fail:**
   - Lista clara de URLs do sitemap sem route correspondente (com sugestão "rota provavelmente removida em refactor; remover do sitemap OU criar redirect 301")
   - Lista clara de routes indexáveis sem entry em sitemap (com sugestão "adicionar em `app/sitemap.ts` ou marcar como noindex se intencional")
   - Exit code 1 em fail; 0 em pass

6. **Exclusões config-driven:**
   - `scripts/sitemap-routing.config.json` lista paths exclusos:
     ```json
     {
       "excludeFromSitemap": ["/studio", "/api"],
       "excludeFromRoutingCheck": ["/preview-secret-route"],
       "noindexRoutes": ["/internal/test-page"]
     }
     ```
   - Mudanças nesta config requerem PR com aprovação @qa

7. **Tests unitários** `scripts/__tests__/validate-sitemap-routing.test.ts`:
   - Mock 5 cenários:
     - Sitemap = Routes (consistent) → pass
     - Sitemap missing 1 URL → fail com URL listada
     - Sitemap has extra URL → fail
     - Dynamic segment com generateStaticParams expand corretamente
     - noindex route excluída corretamente

8. **Tests integration** `tests/integration/sitemap-routing.test.ts`:
   - Build real Next.js + run script
   - Validate em fixture project com 5 routes + 3 sitemap entries
   - Assert exit code + output

9. **Performance:**
   - Run completo <2min em CI free tier (após cache `.next/`)
   - Para projeto com 1.500+ URLs (Wave 1 + Epic 7): aceitável até 5min (não bloqueia >10 min)

10. **Documentação** `docs/architecture/ci-sitemap-routing-strategy.md`:
    - Como o script funciona
    - Como adicionar exclusão (raro, justificado)
    - Como remediar fail comum (rota removida sem update sitemap)
    - Cross-reference NFR26 + Story 1.8

11. **NFR26 alinhamento** verificado em PR DoD checklist:
    - Run em PR de teste (criar PR com inconsistência intencional → build deve falhar)
    - Run em PR consistente → build passa
    - Screenshot do GitHub Actions output anexado em DoD

12. **Bloqueio merge real:**
    - Branch protection rule no GitHub configurada (Story 1.3 ou agora) — "Required status check: validate-sitemap-routing"
    - Verificar via test: tentar merge sem o status check → bloqueado

13. **Cross-validation com Story 7.10 pipeline mensal:**
    - Após pipeline mensal de rede credenciada (Story 7.10) commit auto, esta validação roda — se nova rota aparecer (ex: prestador novo) sem sitemap update, fail
    - Story 7.10 deve atualizar sitemap.ts no mesmo commit

14. **Compatibility com `next-sitemap` ou similar:**
    - Caso projeto migre de `app/sitemap.ts` nativo para `next-sitemap` plugin no futuro, script abstrai source via adapter pattern
    - Documenta adapter em `validate-sitemap-routing.mjs`

15. **Anti-padrão concorrente coberto:**
    - Story DoD inclui screenshot/anotação demonstrando que o pattern duplo de `amilsaudebr.com.br` (`/sao-paulo` + `/tabela-amil-sao-paulo-2026.html`) seria detectado e bloqueado pelo nosso CI
