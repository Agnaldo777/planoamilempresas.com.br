## Story

**As a** @dev,
**I want** implementar tipos novos + loader compatível com schema redesenhado do dataset (`prestadores[].redes: string[]`),
**so that** todas as Stories 7.2-7.10 (hub, prestador, cidade, bairro, Cluster E, tipo, schemas, pipeline) tenham fundação tipo-segura, com cache em memória e zero literal "Bradesco" em código novo.

## Strategic Context / Why

O loader atual `data/rede-credenciada/rede-amil.ts` (linhas 62-79) espera schema raw COMPLETAMENTE DIFERENTE do dataset 2026-04-26:

- **Loader esperava:** `{Prestador (combinado), 10 booleanos por rede}` — 14 colunas
- **Dataset novo:** `{codigo, nome, uf, municipio, bairro, redes: string[]}` — 6 fields, redes como array

Pax flagou isto como **BLOCKER B-01** no `po-revalidation-rede-credenciada-integration.md`: 5 redes do enum antigo sumiram, 6 novas apareceram. SCP v1.2.3 §4.3 Story 7.1 re-tamanha de S (2-3d) para M (4-5d) com AC explícita de rewrite. Esta story é **foundation de todo Epic 7** — sem ela, todas as outras stories ficam bloqueadas.

## Acceptance Criteria

1. Novo arquivo `src/types/rede-credenciada-amil.ts`:
   - `interface PrestadorAmilRaw { codigo: string; nome: string; uf: string; municipio: string; bairro: string; redes: string[] }`
   - `interface PrestadorAmil extends PrestadorAmilRaw { tipoInferido: TipoAtendimentoInferido; slug: string }`
   - `interface DatasetRedeAmil { geradoEm: string; fonte: string; totalRedes: number; totalPrestadores: number; redes: string[]; prestadores: PrestadorAmilRaw[] }`
   - `type TipoAtendimentoInferido = 'Hospital' | 'Pronto-Socorro' | 'Maternidade' | 'Clínica' | 'Laboratório' | 'Diagnóstico por Imagem' | 'Centro/Instituto' | 'Odontologia' | 'Outro'`
   - `type RedeAmilNome = '...'` (11 strings das redes ativas — confirmadas em Story 7.0a)
   - Type guard runtime: `isDatasetRedeAmil(x: unknown): x is DatasetRedeAmil`

2. Novo arquivo `src/lib/operadoras/amil/rede-credenciada-loader.ts` com 13+ helpers:
   - `getAllPrestadores(): PrestadorAmil[]`
   - `getMunicipios(): MunicipioRedeAmil[]`
   - `getMunicipioBySlug(ufSlug: string, cidadeSlug: string): MunicipioRedeAmil | null`
   - `getPrestadoresPorMunicipio(ufSlug: string, cidadeSlug: string): PrestadorAmil[]`
   - `getPrestadoresPorBairro(ufSlug: string, cidadeSlug: string, bairroSlug: string): PrestadorAmil[]`
   - `getMunicipiosByUf(uf: string): MunicipioRedeAmil[]`
   - `getTopMunicipios(limit: number): MunicipioRedeAmil[]`
   - `getBairrosDoMunicipio(prestadores: PrestadorAmil[]): { bairro: string; total: number }[]`
   - `getPrestadoresPorRede(rede: RedeAmilNome): PrestadorAmil[]`
   - `getPrestadoresPorTipo(tipo: TipoAtendimentoInferido): PrestadorAmil[]`
   - `getEstatisticasRede(): EstatisticasRede`
   - `getEstatisticasByUF(uf: string): EstatisticasUFAmil`
   - `prestadorSlug(p: PrestadorAmil): string`
   - `getPrestadorBySlug(uf: string, cidadeSlug: string, prestadorSlug: string): PrestadorAmil | null`

3. `inferTipoAtendimento(nome: string): TipoAtendimentoInferido` — preserva 8 categorias do loader antigo + adiciona "Odontologia" + "Pronto-Socorro" enriquecidos. Regex testados unitariamente.

4. Cache em memória (`Map`) construído no boot do build worker:
   - Chaves principais: `byUf`, `byMunicipioSlug`, `byBairroSlug`, `byPrestadorSlug`, `byRede`, `byTipo`, `municipios`, `estatisticas`
   - Sem chamadas runtime — tudo build-time
   - `parsePrestador()` enriquece raw → adiciona `tipoInferido` e `slug`

5. Slug helper determinístico:
   - `slugify(value: string): string` (NFD normalize + lowercase + non-alnum → `-`)
   - Idempotente: `slugify(slugify(x)) === slugify(x)`
   - Tests: 6 casos canônicos (acentos, espaços múltiplos, símbolos, vazio, números, UTF-8)

6. Deprecar `data/rede-credenciada/rede-amil.ts`:
   - Header com `@deprecated — substituído por src/lib/operadoras/amil/rede-credenciada-loader.ts (Story 7.1)`
   - Re-exports apontando para o novo path para compatibilidade temporária
   - **Remover completamente em Story 7.2** (após validação que ninguém mais consome)

7. Tests unitários (`src/lib/operadoras/amil/__tests__/rede-credenciada-loader.test.ts`):
   - `getAllPrestadores()` retorna array com `length === 9325` (validação contra dataset commitado)
   - `getMunicipios()` retorna 438 municípios únicos
   - `getEstatisticasRede().totalUFs === 26`
   - `getPrestadoresPorRede('AMIL S750 QP')` retorna 4959 prestadores (count exato)
   - `inferTipoAtendimento('HOSPITAL DAS CLINICAS') === 'Hospital'`
   - `inferTipoAtendimento('LABORATORIO ALPHA') === 'Laboratório'`
   - Idempotência de `slugify`

8. Imports absolutos `@/types/rede-credenciada-amil` e `@/lib/operadoras/amil/rede-credenciada-loader` — alinhado com NFR14 + `code-standards.md` ("Sempre absolutos. Nunca relativos.")

9. `pnpm build` (Next.js) passa sem erro TypeScript em strict mode

10. Lint passa em strict (`eslint --max-warnings=0`)

11. **NFR8 enforcement:** zero literal "Bradesco" em código novo (`grep -r 'Bradesco' src/lib/operadoras/amil/` retorna 0)

12. **CI gate:** Story 7.1 PR não pode mergir sem (a) testes verdes, (b) build verde, (c) lint verde, (d) coverage ≥80% no loader.
