## Story

**As a** decisor PJ avaliando cobertura geográfica da rede Amil,
**I want** uma página hub `/rede-credenciada` com busca livre filtrável (cidade, bairro ou prestador), atalhos por estado, filtros avançados de tipo/rede, FAQ rica e estatísticas dataset-driven,
**so that** eu localize prestadores em ≤3 cliques sem sair da página, com ≤30KB de JS de busca, e o site capture buscas head-term ("rede credenciada amil", "hospitais amil") com FAQPage schema rich.

---

## Strategic Context / Why

Story 7.2 é o **Ataque 1 dos 5 ataques SEO** do Epic 7 (PRD §Ataque 1, FR13+FR23) e a **porta de entrada de toda a rede credenciada**. Sem hub funcional, todas as ~10.500 URLs SSG das stories 7.4-7.8 ficam órfãs de internal linking e perdem o moat de autoridade de página.

Três motivações específicas:

1. **Substitui o componente legado deprecado em 7.1.** O `data/rede-credenciada/rede-amil.ts` foi marcado `@deprecated` em 7.1; Story 7.2 é a primeira consumidora do novo loader `src/lib/operadoras/amil/rede-credenciada-loader.ts` e remove o legado em definitivo (AC11).
2. **Endereça o BLOCKER B-01 da @po Pax** (`po-revalidation-rede-credenciada-integration.md` §74-76): o `<NetworkSearch />` precisa rodar com index MiniSearch ≤30KB para honrar NFR2 (≤100KB JS total) — caso contrário, perdemos o orçamento de bundle do site inteiro só na página da rede.
3. **FE Spec drift conhecido** (Pax revalidation report v1.2.2 §160-166): o FE Spec v1.0 citava "endereço + especialidades" que **não existem no dataset Power BI**. Uma já patchou para v1.1 (Screen 7 atualizada — `front-end-spec.md` linhas 732-789) e essa story é a primeira a implementar Screen 7 v1.1 *as-is* (sem reintroduzir os campos fantasma).

## Acceptance Criteria

1. **Rota hub criada:** `app/(marketing)/rede-credenciada/page.tsx` é um React Server Component com `export const revalidate = 604800` (ISR 7 dias) e `export const dynamic = 'force-static'`. RSC consome `getEstatisticasRede()` e `getTopMunicipios(5)` do loader (AC do 7.1) — zero fetch runtime.
2. **`<NetworkSearch />` Client Component** em `src/components/network/NetworkSearch.tsx` (`'use client'`) com index MiniSearch pré-built importado como JSON estático. Bundle gzipado do componente + index ≤ **30KB** (medir via `next build` + `webpack-bundle-analyzer`); falha o gate se exceder.
3. **Filtros progressivos:**
   - Busca livre primária via Radix `Command` (`@radix-ui/react-popover` + headless command pattern) — `aria-autocomplete="list"`, debounce 120ms, `useDeferredValue` para INP ≤200ms.
   - Accordion (Radix) com 2 grupos: `Tipo de prestador` (chips multi-select de 8 categorias do `TipoAtendimentoInferido`) + `Rede/Produto Amil` (combobox das 11 redes ativas do enum `RedeAmilNome`).
   - **SEM** campo "Especialidade" (gap conhecido — confirmado em `front-end-spec.md` linha 761 e ADR-007).
4. **Atalhos por estado** via `<UfShortcutChips />` em `src/components/network/UfShortcutChips.tsx`: chips horizontais com top-5 UFs por densidade (`getEstatisticasRede().porUF`), formato `[RJ 3.696]` `[SP 2.996]` `[DF 447]` `[PR 394]` `[MG XXX]` + chip `[+22]` linkando para um futuro `/rede/[uf]` (out of scope desta story; chip apenas leva a uma navigation drawer com lista das 26 UFs).
5. **FAQPage schema JSON-LD** com **exatamente 8 perguntas** renderizadas via `<RedeCredenciadaFAQ />` em `src/components/network/RedeCredenciadaFAQ.tsx`. Conteúdo das 8 perguntas vem de `src/content/rede-credenciada-faq.ts` (constante, sem CMS — Sanity entra apenas em Story 5.x). Validação: `Schema.org` validator + Google Rich Results Test em CI (mesmo gate do canary 1.4).

5-bis. **`<OrganizationJsonLd />` primitive — ADR-006 mitigação 2 (NOVO via amendment O-2):**
   - Criar em `src/components/schema/OrganizationJsonLd.tsx`
   - Renderiza JSON-LD `Organization` exato do ADR-006 §"Schema markup":
     ```jsonld
     {
       "@context": "https://schema.org",
       "@type": "Organization",
       "name": "BeneficioRH (corretor)",
       "sameAs": ["https://www.amil.com.br"]
     }
     ```
   - **Renderizado no hub `/rede-credenciada` desta story** (componente injetado via Next 14 metadata API ou `<script type="application/ld+json">` em RSC).
   - **Primitive reusável transversal:** Stories 7.4 (páginas-prestador), 7.6 (páginas-bairro), 7.7 (Cluster E) e 7.8 (tipo×UF×município) consomem este componente para honrar mitigação 2 do ADR-006 em **todas as páginas que usam termo "Amil" em URL** (~10.500 URLs total). DRY enforced — uma única source-of-truth do schema.
   - **Por que aqui (e não em 7.7):** ADR-006 Context (linhas 22-24) declara escopo transversal — *"Combinado com URLs UF/município/bairro (~10.000+ URLs), o site terá uso massivo de marca Amil em URLs"*. Centralizar no hub (primeira página da Epic 7) garante que stories downstream consomem sem dependência circular com 7.7 (que está bloqueada por ADR-006 Accepted).
   - **Type safety:** props `name?: string` (default `'BeneficioRH (corretor)'`) e `sameAs?: string[]` (default `['https://www.amil.com.br']`) — overrides explícitos para tests, mas valor canônico é o default.
   - **Test:** parse JSON-LD output → validar shape Schema.org `Organization` + presença literal de `https://www.amil.com.br` em `sameAs`.
6. **Stats dataset-driven** (não hardcoded): "9.325 prestadores · 26 UFs · 438 municípios · 11 redes ativas" lido em build-time via `getEstatisticasRede()`. Test snapshot garante que se o dataset mudar (Story 7.10 mensal), os números atualizam sem code change.
7. **Disclaimer compliance** ANS 326305 + corretor SUSEP fixos em footer da página + topo do `<NetworkSearch />` (mitigação 1 do ADR-006). Texto canônico em `src/content/disclaimers/amil-rede.ts` para reuso pelas demais stories Epic 7.
8. **Performance gates (NFR1 + NFR2):**
   - Lighthouse Mobile (Moto G Power perfil): Performance ≥ 92, SEO 100, A11y ≥ 95, Best Practices ≥ 95
   - LCP ≤ 2.0s, INP ≤ 200ms, CLS ≤ 0.05 medidos em CI via Lighthouse CI assertions
   - Bundle JS total da página ≤ 100KB gzip (NFR2)
9. **Acessibilidade WCAG AA:**
   - `aria-autocomplete="list"` no Command
   - Live region (`role="status"` `aria-live="polite"`) anunciando count de resultados ("32 prestadores encontrados em São Paulo")
   - Foco visível em todos os controles (tab order: busca → atalho UF → accordion → resultado → CTA)
   - Test automático via axe-core: zero violations de severidade `serious` ou `critical`
10. **Sitemap entry:** adicionar `/rede-credenciada` ao `app/sitemap.ts` com `priority: 0.9`, `changefreq: 'weekly'`, `lastmod` = `dataset.geradoEm`.
11. **Remoção do legado:** deletar `data/rede-credenciada/rede-amil.ts` (deprecação iniciada em 7.1 AC6) + grep no repo confirmando zero imports remanescentes (`grep -r "from.*data/rede-credenciada/rede-amil"` retorna 0).
12. **CI gate:** PR não mergia sem (a) build verde, (b) lint:strict verde, (c) bundle size guard ≤ 30KB para `<NetworkSearch />` chunk, (d) Lighthouse CI passando os 4 thresholds, (e) axe-core verde, (f) Schema.org validator verde.

### Key Technical Context

**MiniSearch index strategy (NFR2 budget):**
- 9.325 prestadores × ~5 fields médios → naive index ~150-200KB descomprimido
- Comprimido (gzip) tipicamente ~25-35% → 50-70KB. **Excede 30KB.**
- Mitigações ordenadas:
  1. Dropar field `redes` do index (lookup pós-match via `getPrestadorBySlug`)
  2. Dropar `tipoInferido` do index (filtro pós-match)
  3. Truncar `codigo` para id curto sequencial
  4. Se ainda exceder: dividir por UF (lazy-load do shard da UF clicada) — fallback caro mas viável
- **Decisão Aria (esperada no review):** começar com fields = `[id, nome, municipio, bairro]` somente; tipo/rede aplicados via filtro client-side pós-busca usando lookup tabela.

**RSC vs Client boundary:**
- `page.tsx` = RSC; renderiza estatísticas, hero, FAQ (HTML estático), disclaimer.
- `<NetworkSearch />` é a única ilha Client (`'use client'`).
- FAQPage schema injetado via Next 14 metadata API (RSC; zero JS).

**Radix Command pattern:**
- Headless `cmdk`-style (Radix oficial) — não usar libs como `react-select` (>20KB sozinhas).
- `aria-autocomplete="list"` + `aria-controls` apontando para listbox.
- Live region renderiza count em RSC inicial ("9.325 prestadores disponíveis"), atualizada pelo Client após query.

**Bundle budget enforcement no CI:**
- Adicionar `@next/bundle-analyzer` em modo `--analyze`
- Step CI extrai size do chunk `network-search` via grep no output e falha se >30KB
