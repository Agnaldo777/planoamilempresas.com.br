## Story

**As a** @devops,
**I want** conectar o repositório `planoamilempresas` ao Cloudflare Workers (via `@opennextjs/cloudflare@1.19.4`, ADR-011) com preview automático em PRs e DNS Cloudflare apontando para o Worker route do domínio principal,
**so that** cada PR seja deployado em URL única isolada para review e o domínio principal `planoamilempresas.com.br` esteja servindo HTTPS estável desde o primeiro canary.

---

## Strategic Context / Why

Esta é a primeira story de infraestrutura runtime — sem hosting + DNS, nenhuma página é servível publicamente. **ADR-011 Accepted (2026-04-29, co-signed pelo stakeholder)** migrou o deploy target de Vercel para Cloudflare Workers via `@opennextjs/cloudflare@1.19.4` (não Pages — ver ADR-011 §rationale). **ADR-004 (DNS Strategy)** é mantido em essência: Cloudflare DNS com TTL 300s para permitir rollback 301 em <1h (mitigação 4 da Story 1.0 Bloco 2, risco marca Amil), agora apontando para Workers route ao invés de IPs Vercel. Upstash Redis continua provisionado nesta story porque o codebase forkado já depende dele (rate limiting de formulários, cache CNPJ — herdado do clone Bradesco em Story 1.1). Adicionalmente, **R2 bucket** é provisionado para incremental cache (substitui Vercel ISR cache).

Bloqueia toda story que precise de URL pública (1.4 canary, 1.5 analytics, 1.6 LGPD, etc.).

## Acceptance Criteria

(Fonte: PRD v1.2.2 Epic 1 Story 1.2, 11 ACs — ACs reescritas em termos Cloudflare Workers conforme ADR-011)

1. Conta Cloudflare ativa (já existe — ver ADR-004 e projeto Bradesco que opera em Cloudflare Workers) com projeto Workers `planoamilempresas` criado e conectado ao repositório GitHub via `cloudflare/wrangler-action@v3`
2. Deploy automático do branch `main` para production via workflow `cloudflare-workers-deploy.yml` (action `cloudflare/wrangler-action@v3` com command `deploy`)
3. Preview deploy automático em cada PR (Cloudflare Pages preview URL gerada por `wrangler versions upload` + comentada no PR pelo workflow)
4. Build command `npx @opennextjs/cloudflare build` validado em `wrangler.jsonc` + `open-next.config.ts` — Next.js 14 buildando para Workers runtime sem erros
5. Environment variables/secrets configurados em 3 environments (development / preview / production) via Cloudflare Dashboard ou `wrangler secret put`:
   - `NEXT_PUBLIC_GA4_ID`, `GSC_VERIFICATION`, `SENTRY_DSN`, `NEXT_PUBLIC_CLARITY_ID`
   - `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`
   - `CRM_PROVIDER=clint`, `CRM_API_KEY` (placeholder até Story 4.3)
   - `RECEITA_API_KEY` (opcional fallback BrasilAPI)
   - `NEXT_PUBLIC_TURNSTILE_SITE_KEY`, `TURNSTILE_SECRET_KEY`
   - `NEXT_PUBLIC_WHATSAPP_NUMBER=5511926510515`
   - **Novos (ADR-011):** `CF_ENV` (development/preview/production), `CF_VERSION_METADATA` (binding), `GITHUB_SHA` (injetado pelo workflow para tracking)
   - Valores reais aplicados em Story 1.5 (analytics) / 4.3 (CRM)
6. Branch protection no GitHub: PRs obrigatórios para `main`, ao menos 1 approval, CI verde + required check `cloudflare-workers-deploy.yml` (preview deploy) OK
7. DNS Cloudflare configurado: `planoamilempresas.com.br` (A/AAAA) e `www` (CNAME) apontando para Workers route do projeto; **TTL 300s** (5min) para permitir rollback rápido (ADR-004 mantido)
8. HTTPS forçado pelo Cloudflare proxy (automático para zonas Cloudflare) + HSTS via `_headers` file — verificável em curl `-I` retorna `Strict-Transport-Security`
9. Preview deploys protegidos por Cloudflare Access (password/SSO) ou abertos conforme preferência confirmada do stakeholder
10. **R2 bucket** `planoamilempresas-isr-cache` provisionado para incremental cache (substitui Vercel ISR — bind em `wrangler.jsonc`) + **Upstash Redis** KV provisionado (free tier MVP) para rate limiting + cache CNPJ — testado em endpoint placeholder
11. **Cloudflare Web Analytics** habilitado no projeto (substitui Vercel Analytics — tier gratuito incluído na conta Cloudflare)

### Key Technical Context

**ADR-004 (DNS Strategy):** Cloudflare DNS apontando para Vercel é a opção escolhida (Option A), preferível ao Cloudflare Proxy (Option B) porque Vercel já oferece edge network próprio + Vercel Analytics não funcionam corretamente com Cloudflare Proxy ativo. TTL 300s é constraint da mitigação 4 da Story 1.0 Bloco 2 (rollback 301 marca Amil em <1h).

**Upstash Redis:** Free tier suporta 10K comandos/dia + 256MB; suficiente para MVP (rate limiting + cache CNPJ). Região recomendada: São Paulo (latência <50ms para usuários BR) ou US-East se SP indisponível.

**Vercel tier:** Hobby pode bastar para MVP; upgrade para Pro ($20/mês) se precisar de Preview Password Protection ou bandwidth >100GB/mês.
