## Story

**As a** SEO long-tail extremo (hyper-geo),
**I want** ~700-800 páginas estáticas `/rede/[uf]/[municipio]/[bairro]/` filtradas (apenas bairros com ≥3 prestadores classificados), com lista de prestadores + 3 FAQs específicos do bairro + sitemap shard próprio,
**so that** capturamos buscas hyper-localizadas ("hospital amil copacabana", "rede credenciada amil tatuapé") onde concorrentes não cobrem 1/3 do volume, sem indexar páginas thin (1-2 prestadores → noindex + canonical para cidade-pai).

---

## Strategic Context / Why

Story 7.6 é o **Ataque 4** do Epic 7 e o de **menor concorrência mapeada** (`competitor-analysis.md` — zero concorrentes Amil cobrem páginas-bairro). Três motivações:

1. **Long-tail moat sem competição.** Buscas como "hospital amil copacabana" e "laboratório amil pinheiros" têm volume baixo mas alta intenção (decisor já está no bairro pesquisando). Concorrentes ficam em head-term ou cidade — bairro é gap aberto.
2. **Filtro anti-thin é mandatório** (Helpful Content + audit Story 5.7). Dataset bruto tem **1.687 bairros** mas a maioria com 1-2 prestadores. Apenas ~700-800 bairros têm ≥3 prestadores — o resto vai para noindex + canonical para `/rede/[uf]/[municipio]/`.
3. **Hyper-geo schema enriquece SERP.** Schema `MedicalOrganization` com `areaServed` específico do bairro + `BreadcrumbList` 4-níveis (Home › Cidade › Bairro › Lista) é diferenciador rich snippet.

## Acceptance Criteria

1. **Rota dinâmica SSG** `app/(marketing)/rede/[uf]/[municipio]/[bairro]/page.tsx` com `generateStaticParams()` retornando **TODOS os bairros** com `≥1 prestador` (~1.687 combinações). A diferenciação entre "indexável" vs "noindex+canonical" acontece via `generateMetadata` consumindo o threshold (AC2/AC3) — não via filtro do `generateStaticParams`.
2. **Filtro anti-thin determinístico (binário index vs noindex):**
   - Helper `getBairrosViaveisPorCidade(uf, municipio): { bairro, slug, total }[]` retorna só `total >= 3` (usado em sitemap AC8 + chips de internal linking AC7)
   - Helper `getTodosBairrosPorCidade(uf, municipio): { bairro, slug, total }[]` retorna TODOS os bairros com `total >= 1` (usado em `generateStaticParams` AC1)
   - Threshold `MIN_PRESTADORES_BAIRRO = 3` em `src/config/seo.ts` (compartilhado com 7.8) — separa "indexável" (≥3) de "noindex+canonical" (1-2)
   - Test snapshot: total de páginas geradas em `[1500, 1700]`; total **viáveis indexadas** em `[700, 800]`; total **noindex+canonical** em `[700, 1000]`
3. **NoIndex + canonical para bairros 1-2 prestadores (literal SCP v1.2.3 §4.3 AC3):**
   - **Inclui** em `generateStaticParams` — bairros sub-3 geram página SSG com `<meta name="robots" content="noindex,follow">` em `<head>` (Next 14 metadata API: `metadata.robots = { index: false, follow: true }`)
   - **Canonical aponta para cidade-pai:** `<link rel="canonical" href="/rede/[uf]/[municipio]/">` (Story 7.5)
   - Conteúdo da página é renderizado normalmente (mesmo template do AC4) — Googlebot pode crawlear, mas não indexa; segue links internos preservando link equity
   - **Razão (process adherence O-3 Caminho A):** preserva link equity de backlinks externos para `/rede/sp/sao-paulo/algum-bairro-pequeno/` (Googlebot ainda visita → segue links para `/rede/sp/sao-paulo/` e prestadores). 404 + 301 catch-all teria 2 hops (perda PageRank ~10%) — descartado.
   - **Compatível com `force-static` (AC9)** — toda a rota é SSG, sem `force-dynamic`
   - Test fixture: bairro com 2 prestadores → render OK + meta robots noindex,follow + canonical correto
4. **Conteúdo da página:**
   - H1: "Rede Credenciada Amil em {Bairro} — {Município}/{UF} ({N} Prestadores)"
   - Lista de prestadores com cards (`<NetworkResultCard />` reusado da 7.2)
   - 3 FAQs específicos (template parametrizado: "Quantos hospitais Amil em {Bairro}?", "Quais redes Amil aceitas em {Bairro}?", "Como agendar consulta em prestador Amil em {Bairro}?")
   - Disclaimer canônico (import de `src/content/disclaimers/amil-rede.ts`)
5. **Schema.org JSON-LD:**
   - `BreadcrumbList` 4 níveis (Home › Cidade › Bairro)
   - `FAQPage` com 3 perguntas
   - `ItemList` com prestadores top-N (limit 20)
   - `MedicalOrganization` array com `address.addressLocality` = município, `address.addressRegion` = UF, `areaServed` = bairro

5-bis. **`<OrganizationJsonLd />` reuso transversal — ADR-006 mitigação 2 (NOVO via amendment O-2):**
   - Importar `<OrganizationJsonLd />` de `@/components/schema/OrganizationJsonLd` (criada em Story 7.2)
   - Renderizar em **todas as 700-800 páginas-bairro viáveis**
   - Render usa props default canônicos (`name: 'BeneficioRH (corretor)'`, `sameAs: ['https://www.amil.com.br']`)
   - **Razão:** páginas-bairro contêm UF/município/bairro em URL com termo "Amil" implícito (rede credenciada Amil) → ADR-006 mitigação 2 aplicável transversalmente
   - **Test:** smoke E2E em 5 URLs estratificadas (AC10) verifica presença de `<script type="application/ld+json">` `Organization` com `sameAs: amil.com.br`

6. **`generateMetadata`:**
   - Title: "Rede Credenciada Amil {Bairro} — {Município}/{UF} | {N} Prestadores"
   - Description (~155 chars): bairro + total + tipo dominante + redes principais
   - Canonical: `/rede/[uf]/[municipio]/[bairro]/`
7. **Internal linking:**
   - Breadcrumb com BreadcrumbList JSON-LD
   - Link "Ver toda rede em {Município}" → `/rede/[uf]/[municipio]/` (Story 7.5)
   - Link "Outros bairros de {Município}" → drawer com top-10 bairros viáveis da cidade
   - Cards de prestadores linkam para `/rede/[uf]/[municipio]/[prestadorSlug]/` (Story 7.4)
8. **Sitemap shard** `app/sitemap-rede-bairros.xml/route.ts`:
   - Apenas as 700-800 URLs viáveis (≥3 prestadores)
   - `<lastmod>` = `dataset.geradoEm`
   - Priority 0.6 (long-tail mas alta intenção)
   - changefreq `weekly`
9. **ISR revalidate** = `2592000` (30 dias) + `force-static`.
10. **Performance gates (NFR1):** amostra de 5 páginas-bairro estratificadas (1 cauda denso top-3 + 1 mid-tier + 3 cauda mais fina) Lighthouse CI: Perf ≥92, LCP ≤2.0s, INP ≤200ms, CLS ≤0.05.

### Key Technical Context

**Threshold = 3 rationale:**
- 1-2 prestadores = thin (Helpful Content flag risk)
- 3+ permite mínimo de variação (1 hospital + 1 lab + 1 clínica = relevância)
- Compartilhado com Story 7.8 (tipo×UF×município)
- Configurável em `src/config/seo.ts` para tuning pós-launch

**Decisão noindex+canonical 200 (O-3 Caminho A — SCP §4.3 literal):**
- Bairros sub-3 renderizam página completa (200) com `<meta name="robots" content="noindex,follow">` + `<link rel="canonical" href="/rede/[uf]/[municipio]/">`
- Razão (preferida sobre 404): preserva link equity de backlinks externos. Googlebot ainda visita, não indexa, mas segue links internos para cidade-pai e prestadores → contribui PageRank para URLs-valor.
- 404 + 301 catch-all em middleware foi alternativa avaliada e descartada em revalidation O-3 (`po-revalidation-epic7-stories-7.2-7.8.md` MAJOR-01) — penaliza ~10% PageRank em URLs com backlink (2 hops vs 1).
- **Trade-off aceito:** build budget cresce de 700-800 para ~1.687 páginas (+~5-7s no build). Aceitável dentro do budget Phase 1 ≤25min do Architecture §6.1.
- **Sitemap shard (AC8) só lista os ~700-800 viáveis** — Googlebot não é convidado a crawlear sub-3 explicitamente, mas se chega via backlink, recebe 200 noindex+canonical (SEO-friendly).

**Slug ambiguidade:**
- "CENTRO" existe em 50+ cidades; slug do bairro sozinho colide
- Path completo `/rede/[uf]/[municipio]/[bairro]/` desambigua
- Helper `bairroSlug(uf, municipio, bairro)` já no loader 7.1 deve incluir contexto

**Anti-thin coverage report (auto-gerado mensal):**
- Story 7.10 pipeline gera novo report a cada update do dataset
- Report stored em `docs/audit/rede-bairros-coverage-{YYYY-MM}.md`
- Útil para detectar drift (ex: cidade que tinha 800 bairros viáveis cai para 750)
