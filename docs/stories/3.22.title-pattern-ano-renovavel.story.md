## Story

**As a** decisor PJ vendo SERP "plano amil empresarial 2026" e **@po (Pax)**,
**I want** title pattern `[Tipo] Amil [ANO] — [Qualificador]` em todas páginas indexáveis com ano injetado dinamicamente via Sanity v3 single-source-of-truth `currentYear` document, consumido por Next.js `generateMetadata()` em cada `page.tsx`,
**so that** o satélite cubra FR50 do PRD v1.3.1, sinalize frescura ao Google e ao usuário em SERP, e elimine retrabalho anual de manual rewrite de 1.000+ titles em transição 2026→2027.

---

## Strategic Context / Why

`amilsaudebr.com.br` (research `09-amilsaudebr-kitcorretor-deep-scrape.md`) usa "Amil 2026" no title — gap competitivo barato cobrir. Frescura em SERP = CTR uplift potencial + sinal de relevância para Google.

Sem variável SSOT em Sanity, transição anual (2026→2027) requer **rewrite manual de N titles em N páginas** — drift garantido em 1.500+ páginas (cornerstones, pillar, programmatic, rede, comparadores, blog). Com `currentYear` Sanity:

1. Stakeholder atualiza um único document em janeiro
2. Vercel ISR revalida todas as páginas dependentes
3. Zero drift, zero retrabalho

ADR-008 (Stack unificada Next.js) já confirma App Router; `generateMetadata()` é o entry-point oficial para metadata dinâmico no App Router. ADR-001 confirma Sanity v3 como CMS.

## Acceptance Criteria

1. **Sanity v3 schema document `siteSettings`** (singleton):
   - Field `currentYear: number` (default 2026, validation: integer between 2024 and 2100)
   - Field `nextYearTransitionDate: datetime` (opcional — auto-bump em janeiro)
   - Field `titlePatternPrefix: string` (default "Plano de Saúde")
   - Field `titlePatternSuffix: string` (default "Empresarial PJ")
   - UI Sanity Studio mostra preview do title pattern resolvido

2. **GROQ query helper** `lib/seo/title.ts` (RESOLVIDO 2026-05-01 — Pax: helper consolidado neste path real; projeto não usa estrutura `src/` na raiz para libs):
   - `getSiteSettings(): Promise<SiteSettings>` retorna `{ currentYear, titlePatternPrefix, titlePatternSuffix, ...}`
   - Cache via Next.js `unstable_cache` ou ISR revalidate 3600s

3. **Helper de title `buildTitle()`** `src/lib/seo/title.ts`:
   ```typescript
   buildTitle(opts: { tipo: string; qualificador?: string; cidade?: string; siteSettings: SiteSettings }): string
   // Exemplos:
   // buildTitle({ tipo: 'Tabela', qualificador: 'PME', siteSettings }) → "Tabela Amil 2026 — PME"
   // buildTitle({ tipo: 'Plano de Saúde', cidade: 'São Paulo', siteSettings }) → "Plano de Saúde Amil 2026 — São Paulo"
   ```
   - Garante max 60 chars (NFR11). Se exceder: trunca qualificador inteligentemente
   - TypeScript strict tipado

4. **Integração em `generateMetadata()` por template:**
   - `app/page.tsx` (homepage): "Plano de Saúde Amil 2026 — Empresarial PJ"
   - `app/(marketing)/tabela-precos-amil-2026/page.tsx`: "Tabela Amil 2026 — Preços PME"
   - `app/(marketing)/produtos/[slug]/page.tsx`: "Amil [Produto] 2026 — Cobertura e Preço"
   - `app/(marketing)/[cidade]/page.tsx` programmatic: "Plano de Saúde Amil 2026 — [Cidade]"
   - `app/(marketing)/precos/[uf]/page.tsx`: "Tabela Amil 2026 — [UF Nome]"
   - `app/(marketing)/comparar/page.tsx` (Story 3.21): "Comparar Planos Amil 2026"
   - `app/(marketing)/blog/[slug]/page.tsx`: usa título do post (não usa pattern com ano genérico)
   - Cornerstones: "[Tópico] Amil 2026 — Guia [Qualificador]"

5. **Single source of truth enforcement:**
   - Lint rule custom `eslint-plugin-local/require-buildTitle`: warn se `metadata.title` é string literal contendo ano hard-coded (`/2026|2027/`) sem ser `buildTitle()` output
   - Strict mode em `lint:strict` (CI) bloqueia hard-coded year

6. **Renovação anual automatizada (opcional, gated):**
   - GitHub Action `.github/workflows/yearly-bump.yml`:
     - Cron: 1 de janeiro 00:01 BRT
     - Verifica se `siteSettings.currentYear` está atrasado vs ano corrente
     - Abre PR auto-mergeable atualizando o document via Sanity API
     - Notifica @po para review antes de merge
   - **Default OFF** v1; ativar v2 após primeiro ciclo manual sucesso

7. **Visualização em outras meta tags + JSON-LD:**
   - `<meta name="description">` também recebe ano interpolado quando relevante (FR32)
   - `<meta property="og:title">` espelha o `<title>`
   - JSON-LD `Article.headline` / `WebPage.name` usam o mesmo helper

8. **Tests unitários** `src/lib/seo/__tests__/title.test.ts`:
   - 8 cenários (homepage, tabela, produto, cidade, comparar, cornerstone, fallback sem qualificador, truncate >60 chars)
   - Year mock 2026 + 2027 (smoke test transição)

9. **Tests de integração** `tests/integration/title.test.ts`:
   - 5 URLs SSG renderizam title correto com ano atual
   - Mock Sanity client com `currentYear: 2027` → todas URLs refletem

10. **Migration script** `scripts/seed-site-settings.mjs`:
    - Cria document `siteSettings` no Sanity com defaults
    - Idempotente (skip se já existe)
    - Roda como parte do setup Sanity v3 (Story 3.1)

11. **Documentação** `docs/editorial/year-transition-runbook.md`:
    - Fluxo manual de transição anual (caso GitHub Action OFF)
    - 1 step: stakeholder edita `siteSettings.currentYear` no Sanity Studio
    - Vercel ISR revalida (max 1h)
    - QA spot check 5 URLs

12. **NFR11 enforcement** (title ≤60 chars + único):
    - Validate no build via script `scripts/validate-titles-uniqueness.mjs`
    - Se 2+ páginas tem mesmo title → fail CI
    - Se title >60 chars → warn (não bloqueia, helper trunca)

13. **Performance:**
    - GROQ query de `siteSettings` cached agressivamente (1 query por build)
    - Sem regressão Lighthouse SEO ≥95
    - Sem regressão LCP

14. **Backwards compat:** páginas com `metadata.title` existentes (Story 1.4 canary, etc.) migram para `buildTitle()` em PR único atômico.

15. **Disclaimer in title** evitado — manter title comercial-clean; disclaimers são meta description / footer.
