## Story

**As a** decisor PJ avaliando 2-3 produtos Amil lado-a-lado e **@dev (Dex)**,
**I want** rota dedicada `/comparar/?planos=[slug],[slug]...` com URL shareable, layout 2-3 colunas exibindo specs comparativas (preço, rede, abrangência, coparticipação, acomodação, carências), schema `Product × N`, copy editorial de trade-offs e CTA cotação por produto,
**so that** o satélite cumpra FR38 do PRD v1.3.1, capture queries "amil bronze vs prata", "amil ouro ou platinum", e diferencie do "Solicite Cotação" puramente formulário dos concorrentes oferecendo experiência decisória autônoma.

---

## Strategic Context / Why

Comparativo intra-Amil ("Bronze vs Prata vs Ouro") é **alta intenção comercial** — o decisor já está convencido da Amil e avalia entre tiers. Concorrentes não cobrem este uso (todos focam em "solicite cotação" externalizando a decisão). FR38 + UX paradigma "Escolha por exposição" do PRD ("apresentar opções lado a lado com pros/contras") materializam-se nesta story.

URL shareable (`/comparar/?planos=bronze,prata,ouro`) também é vetor de **internal linking** — cornerstones de produto linkam comparativos relevantes; comparativos linkam de volta a cada produto. Schema `Product` × N + `ItemList` ranqueia em queries comparativas.

Note: FR47 (posts comparativos cross-operadora "Amil vs Bradesco") é **diferente** — é blog post, não esta tool. Esta tool é intra-Amil; FR47 é cross-operadora editorial (Story 6.11).

## Acceptance Criteria

1. **Rota Next.js** `app/(marketing)/comparar/page.tsx`:
   - SSR ou SSG com `searchParams.planos` (Next.js App Router pattern)
   - Slugs aceitos: `bronze`, `bronze-mais`, `prata`, `ouro`, `platinum`, `platinum-mais`, `s380`, `s450`, `s750`, `amil-one-s2500`, `amil-one-s6500`, `black`
   - Mínimo 2, máximo 4 planos por comparação (UX/legibilidade)
   - Query inválida → fallback para `?planos=bronze,prata,ouro` default + banner de aviso

2. **Layout responsivo** (Uma):
   - Desktop: 2-4 colunas lado-a-lado, headers sticky no scroll
   - Mobile: cards empilhados ou tabs (sub-decisão Uma)
   - Linhas de comparação: preço base (faixa 30-39 anos referência), rede credenciada (count prestadores nacional + estado padrão SP), abrangência (regional/estadual/nacional/internacional), coparticipação (sim/não + %), acomodação (QC/QP), carências (mínimas ANS + específicas produto), produtos premium (botão "Sob Consulta" via `<PrecoSobConsulta />` Story 6.1)
   - Diferenças destacadas visualmente (badges "Único do plano X", "Igual aos demais")

3. **URL shareable + canonical:**
   - URL `/comparar/?planos=bronze,prata,ouro` é canônica
   - Ordem dos slugs é normalizada (alphabetic OR by hierarchy) para evitar canibalização (`?planos=ouro,bronze` redirect 301 para `?planos=bronze,ouro`)
   - `<link rel="canonical">` aponta para versão normalizada
   - Sitemap inclui top-10 combinações pré-curated (não todas N×M)

4. **Schema JSON-LD:**
   - `Product` × N (1 por plano comparado)
   - Cada `Product`: `name: "Amil [Slug]"`, `brand: { name: "Amil" }` (apenas em `Product.brand`, NUNCA em `Organization.name` — FR54), `provider: BeneficioRH` (Organization), `offers: { Offer com priceSpecification ou "POA" via PrecoSobConsulta para premium}`
   - `ItemList` agregando os N produtos
   - `BreadcrumbList`: Home → Comparar Planos → [seleção]
   - Validar Rich Results Test screenshot 3 combinações canônicas

5. **Conteúdo editorial dinâmico** ≥400 palavras únicas por combinação canônica (top-10 sitemap):
   - Seção "Para quem cada plano é indicado" — 200 palavras
   - Seção "Trade-offs principais" — 100 palavras
   - Seção "Recomendação por perfil de empresa" — 100 palavras (PME pequena, média, grande)
   - **Cuidado anti-cookie-cutter NFR25:** copy é dinâmica baseada em quais planos comparados (não template repetido)
   - Top-10 combinações canônicas tem human-written copy; combinações ad-hoc renderizam copy genérica

6. **Component `<PlanComparison />`** em `src/components/comparator/PlanComparison.tsx`:
   - Recebe `plans: PlanSlug[]`
   - Renderiza tabela comparativa + bullets editoriais + CTAs
   - Usa `<PriceTable>` (Story 6.1) embedded para preços
   - Usa `<PrecoSobConsulta>` para produtos premium

7. **CTA por produto:**
   - Cada coluna tem botão "Cotar [Produto]" linkando para formulário pré-preenchido OU WhatsApp pré-preenchido com mensagem `"Olá, quero cotação Amil [Produto] para minha empresa"`
   - Analytics event `comparator_cta_click` com produto

8. **Internal linking:**
   - Cornerstones de produto (`/produtos/amil-bronze/` etc.) linkam para `/comparar/?planos=bronze,prata` em CTA "Compare com Prata"
   - Comparativos linkam para cada produto individual + tabela de preços (Story 6.2) + calculadora coparticipação (Story 6.3)

9. **Top-10 combinações pré-curated** (em `src/content/comparador/canonical-comparisons.ts`):
   - Bronze vs Prata — PME entry-level
   - Bronze vs Prata vs Ouro — full upgrade ladder
   - Prata vs Ouro vs Platinum — mid tier
   - Platinum vs Black — premium
   - S380 vs S450 vs S750 — Selecionada line
   - Bronze vs S380 — entry budget cross-line
   - Ouro vs Platinum — mid-premium
   - Black vs Amil One S6500 — top premium
   - Amil One S2500 vs S6500 — One line
   - PME 30 vidas: Bronze vs Prata vs Ouro — perfil específico

10. **Tests unitários** `src/components/comparator/__tests__/PlanComparison.test.tsx`:
    - Render 2 planos
    - Render 4 planos
    - Schema gerado tem N Products
    - URL normalization 3 cenários
    - Plan slug inválido → fallback

11. **A11y WCAG 2.1 AA:**
    - Tabela com `<th scope>`, caption (alinha FR52)
    - Diferenças destacadas semanticamente (não só cor)
    - Foco visível
    - Mobile: tabs com `role="tablist"` correto

12. **Performance:**
    - Lighthouse ≥92
    - LCP <2.0s
    - SSG para top-10 combinações; SSR/edge para ad-hoc

13. **NFR25 alinhamento:**
    - Top-10 combinações têm conteúdo único 400+ palavras
    - Combinações ad-hoc renderizam **copy genérica** + warning UX "comparação básica" + sugestão de visitar combinação curated
    - Story 5.7 audit inclui amostra de 5 comparativos canônicos

14. **Schema validation CI** valida `Product × N` + `ItemList` corretos.

15. **Disclaimer obrigatório:**
    - "Comparativo informativo. Valores e coberturas sujeitos a análise contratual da operadora. Última atualização: [data]."
    - Story 1.0 brand-usage-policy compliance (no logo Amil oficial)
