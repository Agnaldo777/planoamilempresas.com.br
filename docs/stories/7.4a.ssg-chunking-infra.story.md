## Story

**As a** SEO programmatic strategist,
**I want** infraestrutura de chunking SSG pronta — helper `chunked-static-params.ts` para fatiar 9.325 prestadores em Phase 1 (Sudeste 7.166) + Phase 2 (≤2.159), rota dinâmica `/rede/[uf]/[municipio]/[prestadorSlug]/page.tsx` com `generateStaticParams()` filtrando por flag `PHASE_2_ENABLED`,
**so that** o pipeline SSG completo (Story 7.4b) tenha base estável e testada para ativar `BUILD_FULL_PROVIDERS=true` quando deploy Cloudflare Workers (Stories 1.2 + 1.3 v2) estiver verde, sem retrabalho na camada de infra.

---

## Strategic Context / Why

Esta story isola o **risco baixo / valor alto** do split de 7.4:

- **Risco baixo:** infra (helpers + rota dinâmica + tests) não depende de servidor real, pode ser implementada e validada offline (`tsc`, `vitest`, `eslint`).
- **Valor alto:** desbloqueia 7.4b (ativação full pipeline) imediatamente após deploy Cloudflare — sem nova rodada de design/implementação.

Quinn (QA) sinalizou em 2026-05-01 que infra estava **5/14 AC cobertos** (chunking + dynamic route + tests passando) enquanto 9 AC restantes dependem de deploy real. Manter tudo em uma única story L bloqueava o reconhecimento formal do trabalho já feito. Pax aprovou split.

---

## Acceptance Criteria

Esta story herda os seguintes AC da master 7.4 (escopo: infra apenas):

1. **(AC1 master) Rota dinâmica SSG** `app/(marketing)/rede/[uf]/[municipio]/[prestadorSlug]/page.tsx` com `generateStaticParams()` retornando combinações `{uf, municipio, prestadorSlug}`. Phase 1 retorna apenas Sudeste (UFs RJ+SP+MG+ES); Phase 2 atrás de flag.

2. **(AC2 master) Chunking infraestrutural** via `src/lib/operadoras/amil/chunked-static-params.ts`:
   - Phase 1: filtro Sudeste (RJ, SP, MG, ES) → 7.166 páginas
   - Phase 2: filtro complemento → ≤2.159 páginas
   - Toggle via env `PHASE_2_ENABLED` (default `false`)
   - Toggle adicional `BUILD_FULL_PROVIDERS` em CI (default `false` para preservar dev iteration speed)

5. **(AC5 master) ISR revalidate** = `2592000` (30 dias) + `dynamic = 'force-static'` configurados na rota.

8. **(AC8 master) Sitemap shard scaffold** — arquivo `app/sitemap-prestadores.xml/route.ts` esboçado com adapter ao chunking helper (pode ficar gated em 7.4b se preferir; aceita-se commit em qualquer das duas).

### AC adicional desta story

A1. **Tests unitários** `src/lib/operadoras/amil/__tests__/chunked-static-params.test.ts` (7 tests) + `app/(marketing)/rede/[uf]/[municipio]/[prestadorSlug]/__tests__/page.test.tsx` (≥4 tests) — cobertura ≥80% no helper.

A2. **Validações automatizadas verdes:**
   - `npx tsc --noEmit` PASS
   - `npx eslint .` PASS (0 problems)
   - `npx vitest run` — tests da rota + helper passando

A3. **Documentação inline** no helper explicando a discriminação Phase 1 vs Phase 2 + as duas flags (`PHASE_2_ENABLED`, `BUILD_FULL_PROVIDERS`).
