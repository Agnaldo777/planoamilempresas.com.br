## Story

**As a** decisor PJ avaliando "quanto tempo até usar o plano para [procedimento]" e **@dev (Dex)**,
**I want** widget JS interativo `<CarenciaCalculator />` que recebe procedimento + tipo de plano + data início e retorna data prevista de uso, embedável inline no cornerstone de carência (Story 3.X) e em página dedicada `/carencia/calculadora/`,
**so that** o satélite supere `amilplanos.com.br` (que menciona carência só em texto, sem ferramenta — FR39 anti-erro) e ofereça utility tool single-purpose que ranqueia para "calculadora carência amil" + variações.

---

## Strategic Context / Why

Carência é **decisão crítica de timing** — o decisor PJ precisa saber "se contratar hoje, parto cobre quando", "consulta quando", "internação eletiva quando". Concorrentes diretos (todos 3) cobrem carência apenas como texto estático em FAQ — ninguém oferece **calculadora interativa**. FR39 do PRD v1.3.1 transforma este gap em moat.

Custo de implementação é baixo (dataset estático ANS RN 195 + 593/2024 + componente client-side single-page) — retorno é (a) longe-tail "calculadora carência [procedimento] amil" + (b) lead magnet (bottom CTA "salvar simulação por email"), (c) shareability pelo URL com query params.

ADR-003 valida fórmula da Calculadora de Coparticipação (Story 6.7); esta calculadora é **diferente** — não há fórmula atuarial complexa, apenas mapeamento `procedimento → carência mínima ANS + carência específica produto Amil`. Validação primária é **regulatória** (advogado co-sign do dataset) não atuarial.

## Acceptance Criteria

1. **Componente Client `<CarenciaCalculator />`** em `src/components/calculators/CarenciaCalculator.tsx`:
   - 3 inputs: procedimento (select com 15+ opções), produto Amil (select 6 produtos premium), data início contrato (date picker, default hoje)
   - Output: data prevista de cobertura + tooltip explicativo (regra ANS aplicada + carência específica produto + link a fonte)
   - Resultado dinâmico (sem submit button — recalcula on input change)
   - Botão "Compartilhar simulação" gera URL com query params
   - Botão "Salvar por email" coleta email + cria lead CRM tagged `carencia_calculator_lead` (LGPD-compliant)

2. **Dataset carências** `src/data/operadoras/amil/carencias.ts`:
   - 15+ procedimentos: parto, parto cesárea, internação clínica, internação cirúrgica, consulta, exames simples (raio-x, ultrassom), exames complexos (RM, TC), oncologia, cardio (cateterismo, marcapasso), próteses ortopédicas, cirurgia plástica reparadora, fertilização, tratamentos psiquiátricos, fisioterapia, transplante
   - Cada procedimento com `carenciaMinimaANS` (RN 195/2009 + 593/2024) + `carenciaProduto` por produto Amil (Bronze/Prata/Ouro/Platinum/Black/etc.)
   - Fonte regulatória citada inline: artigo da RN, publicação oficial Amil ou ANS, link

3. **Compliance regulatória ANS:**
   - Disclaimer obrigatório no widget: "Carências sujeitas a análise da operadora. Valores ANS são mínimos legais; produto Amil pode aplicar carência maior. Confirmar no contrato."
   - Não fazer promessa absoluta ("garantido X dias") — sempre "carência mínima"
   - Advogado revisor (Story 2.4) co-sign do dataset antes de deploy
   - Link "Veja contrato Amil oficial" no footer do widget

4. **Schema JSON-LD `SoftwareApplication`**:
   - `name: "Calculadora de Carências Amil"`, `applicationCategory: "HealthApplication"`, `operatingSystem: "Web"`, `offers: { price: 0, priceCurrency: BRL }`, `provider: BeneficioRH` (FR54)
   - Em página dedicada: também `WebPage` schema

5. **Página dedicada** `app/(marketing)/carencia/calculadora/page.tsx`:
   - SSG (static shell) + Client component island
   - H1 "Calculadora de Carência Amil 2026"
   - Conteúdo editorial ≥600 palavras explicando carência ANS, diferença carência mínima vs específica, exemplos
   - CTA WhatsApp + formulário no rodapé
   - Linka para cornerstone `/carencia-amil-empresarial/` (Story 3.X)

6. **Embed inline no cornerstone de carência** (Story 3.X):
   - Componente Client island carrega só quando visível (intersection observer ou Suspense)
   - Não bloqueia LCP do cornerstone

7. **URL shareable com query params**:
   - `/carencia/calculadora/?procedimento=parto&produto=prata-qc&data=2026-05-01`
   - Estado serializado nos params; reload reproduz
   - URL preview share via `<meta og:image>` dinâmico (opcional)

8. **Tests unitários** `src/components/calculators/__tests__/CarenciaCalculator.test.tsx`:
   - Cálculo correto para 5 cenários canônicos (parto + Bronze, consulta + Black, oncologia + Prata, etc.)
   - Edge: data início no passado → mostra "carência já cumprida"
   - Edge: produto não tem dataset → fallback para mínima ANS + warning
   - URL params parsing + serialization
   - Date picker locale pt-BR

9. **Tests de schema** validate-schema.mjs estende para `SoftwareApplication`.

10. **Acessibilidade WCAG 2.1 AA:**
    - Inputs com `<label>` correto, `aria-describedby` no resultado
    - Resultado dinâmico anuncia via `aria-live="polite"`
    - Navegação por teclado (Tab + Enter)
    - Foco visível
    - Contraste ≥4.5:1
    - Screen reader test documentado

11. **Performance:**
    - Lighthouse ≥90 em página dedicada (alinha NFR1; tolera 90 vs 92 normal por componente JS island)
    - Bundle size do componente <15KB gzip
    - Lazy load — cornerstone tem componente só carregado on-scroll

12. **Analytics events** (depende Story 1.5):
    - `carencia_calc_input` (procedimento mudou)
    - `carencia_calc_result_viewed`
    - `carencia_calc_share_url`
    - `carencia_calc_save_email`

13. **NFR25 alinhamento:**
    - Conteúdo editorial da página dedicada ≥600 palavras únicas
    - Não é programmatic landing — escapa ao audit cookie-cutter

14. **Documentação** `docs/calculators/carencia-data-source.md`:
    - Como o dataset é mantido
    - Quem revisa (advogado SLA 48h)
    - Onde citar fonte regulatória

15. **Validação dataset CI:**
    - Script `scripts/validate-carencias-dataset.mjs` verifica que cada procedimento tem `carenciaMinimaANS` + ≥1 produto mapeado + `fonte` (URL ou referência) preenchida
    - Fail CI em entrada incompleta

- Carências de operadoras outras (Bradesco, etc.) — escopo Amil

- Calculadora de carência **gestante** com data de parto — story futura mais especializada
- Integration com app Amil oficial — escopo público
- LLM-powered "ask carência" chat — não MVP
