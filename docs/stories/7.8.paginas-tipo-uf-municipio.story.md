## Story

**As a** decisor com busca por tipo + urgência ("hospital amil são paulo", "laboratório amil rio"),
**I want** ~250-350 páginas estáticas `/[tipo]/[uf]/[municipio]/` filtradas (≥3 prestadores classificados por tipo dentro do município), com 8 tipos válidos como tipo-segmento de URL, schema `MedicalOrganization` subtipo + `ItemList`, e sitemap shard próprio,
**so that** capturamos buscas tipo-first (fora do `/rede/...` namespace para diferenciar do Cluster E e dos sub-prestadores), com foco em intent imediato ("preciso de um hospital Amil em SP agora").

---

## Strategic Context / Why

Story 7.8 é o **Ataque 5** do Epic 7 — o de **menor volume** (~250-350 URLs) mas alto retorno por intent. Três motivações:

1. **Namespace separado de `/rede/...` é intencional.** URLs como `/hospital/sp/sao-paulo/` em vez de `/rede/sp/sao-paulo/?tipo=hospital` permitem internal linking limpo desde o hub e diferenciam de Cluster E. Trade-off: rota top-level extra. Ganho: SERPs específicas tipo-first (concorrentes não cobrem este corte).
2. **Filtro ≥3 prestadores classificados garante relevância.** Tipos no dataset cru são heuristic (`inferTipoAtendimento` 9 categorias). Combinações tipo×UF×município com 1-2 = thin + risk de tipo errado (regex falha em ~5% dos casos). ≥3 absorve ruído.
3. **Reuso de patterns 7.6** torna esta a story **mais barata do Epic 7 (S, 3d)**. Estrutura quase idêntica a páginas-bairro: filtro determinístico, schemas similar, sitemap shard, internal linking. Diferença: dimensão de filtro (tipo × UF × município) e `MedicalOrganization` subtipo no JSON-LD.

## Acceptance Criteria

1. **Rota dinâmica SSG** `app/(marketing)/[tipo]/[uf]/[municipio]/page.tsx` com `generateStaticParams()` filtrado por `≥3 prestadores` daquele tipo.
2. **8 tipos válidos** (allowlist em `src/config/seo.ts`):
   - `hospital`
   - `laboratorio`
   - `clinica`
   - `diagnostico-imagem`
   - `maternidade`
   - `pronto-socorro`
   - `odontologia`
   - `centro-instituto`
   - **NÃO incluído:** `outro` (slug genérico não captura busca + thin) — tipo `'Outro'` do loader 7.1 mapeia para `notFound()` aqui
3. **Filtro anti-thin determinístico:**
   - Helper `getTipoUfMunicipios(tipo): { uf, municipio, slug, total }[]` que retorna só `total >= 3`
   - Threshold `MIN_PRESTADORES_TIPO = 3` em `src/config/seo.ts` (separado de `MIN_PRESTADORES_BAIRRO`, mesma constante valor)
   - Test snapshot: total entre 250-350 (faixa empírica do dataset 2026-04-26)
4. **NoIndex / 404 para combinações sub-3:**
   - Não inclui em `generateStaticParams`
   - Acessada direta: `notFound()` (consistente com Story 7.6)
5. **Conteúdo da página:**
   - H1: "{Tipo} Amil em {Município}/{UF} — {N} {Tipo Plural} Credenciados"
   - Lista de prestadores com cards (`<NetworkResultCard />` reusado da 7.2)
   - Intro 80-120 palavras templated por tipo + município (gerado via primitive `<TipoIntroContent />`)
   - 3 FAQs templated específicos do tipo: "Quantos {tipo plural} Amil em {Município}?", "Quais redes Amil aceitam {tipo singular} em {Município}?", "Como agendar atendimento {tipo} Amil em {Município}?"
   - Disclaimer canônico (import de `src/content/disclaimers/amil-rede.ts`)
6. **Schema.org JSON-LD:**
   - `BreadcrumbList` 4 níveis (Home › {Tipo} › {UF} › {Município})
   - `FAQPage` com 3 perguntas
   - `ItemList` com prestadores top-N (limit 25)
   - `MedicalOrganization` array com **subtipo correspondente** (`Hospital` para tipo=hospital; `MedicalLaboratory` para tipo=laboratorio; etc.) — reuso do `<PrestadorJsonLd />` discriminated union da 7.4

6-bis. **`<OrganizationJsonLd />` reuso transversal — ADR-006 mitigação 2 (NOVO via amendment O-2):**
   - Importar `<OrganizationJsonLd />` de `@/components/schema/OrganizationJsonLd` (criada em Story 7.2)
   - Renderizar em **todas as 250-350 páginas tipo×UF×município viáveis**
   - Render usa props default canônicos (`name: 'BeneficioRH (corretor)'`, `sameAs: ['https://www.amil.com.br']`)
   - **Razão:** páginas tipo×UF×município contêm UF/município em URL com termo "Amil" no H1 e meta description → ADR-006 mitigação 2 aplicável transversalmente
   - **Test:** smoke E2E em 5 URLs estratificadas (AC11) verifica presença de `<script type="application/ld+json">` `Organization` com `sameAs: amil.com.br`

7. **`generateMetadata`:**
   - Title: "{Tipo Plural} Amil em {Município}/{UF} | {N} Credenciados"
   - Description (~155 chars): tipo + município + total + redes principais
   - Canonical: `/[tipo]/[uf]/[municipio]/`
8. **Internal linking:**
   - Breadcrumb com BreadcrumbList JSON-LD
   - Link "Ver outros tipos em {Município}" → drawer com 8 tipos (chips, link cinza se sem combinações ≥3)
   - Link "Ver toda rede em {Município}" → `/rede/[uf]/[municipio]/` (Story 7.5)
   - Cards linkam para `/rede/[uf]/[municipio]/[prestadorSlug]/` (Story 7.4)
9. **Sitemap shard** `app/sitemap-tipos.xml/route.ts`:
   - 250-350 URLs viáveis
   - `<lastmod>` = `dataset.geradoEm`
   - Priority 0.7 (intent específico, alta intenção)
   - changefreq `weekly`
10. **ISR revalidate** = `2592000` (30 dias) + `force-static`.
11. **Performance gates (NFR1):** amostra de 5 URLs estratificadas (1 hospital × SP, 1 laboratorio × RJ, 1 clinica × MG mid, 1 odontologia × DF, 1 maternidade × cauda) Lighthouse CI: Perf ≥92, LCP ≤2.0s, INP ≤200ms, CLS ≤0.05.
12. **Reuso obrigatório:**
    - `<NetworkResultCard />` (Story 7.2)
    - `<PrestadorJsonLd />` (Story 7.4)
    - Disclaimer canônico (Story 7.2)
    - `inferTipoAtendimento()` + helpers loader (Story 7.1)
13. **CI gate:** PR não mergia sem (a) build verde, (b) lint:strict, (c) tests verdes, (d) Lighthouse CI 5 amostrais, (e) Schema.org validator OK em 3 amostrais (1 hospital + 1 lab + 1 outro), (f) test snapshot count `[250, 350]`, (g) test que tipo `outro` retorna 404.

export const TIPO_SLUGS_VALIDOS = [
  'hospital',
  'laboratorio',
  'clinica',
  'diagnostico-imagem',
  'maternidade',
  'pronto-socorro',
  'odontologia',
  'centro-instituto',

export type TipoSlugValido = typeof TIPO_SLUGS_VALIDOS[number]

// Mapping inverso (slug → enum loader 7.1)
export const SLUG_TO_TIPO: Record<TipoSlugValido, TipoAtendimentoInferido> = {
