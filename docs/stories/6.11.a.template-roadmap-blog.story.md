## Acceptance Criteria

1. **Sanity schema `blogPost`** estendido em `sanity/schemas/blogPost.ts`:
   - Campos: `title`, `slug` (auto kebab), `excerpt` (max 200ch), `body` (Portable Text com blocos rich), `author` (reference Author — Story 6.9), `reviewedBy` (reference Author opcional), `category` (reference Category), `tags` (array string), `publishedAt`, `updatedAt` (auto), `ogImage` (image + alt obrigatório), `seoTitle` (max 60ch), `seoDescription` (max 160ch)
   - Validation rules em todos os campos obrigatórios
   - Preview no Studio com title + author + publishedAt
   - Verificar campos parciais já existentes — não duplicar, complementar

2. **Página listing** `app/(marketing)/blog/page.tsx`:
   - SSR/ISR fetch posts do Sanity (orderBy `publishedAt desc`)
   - Paginação SSR (12 posts/página, query param `?page=N`) — **NÃO** infinite scroll (motivo: SEO-friendly + crawlable)
   - Componente `<BlogListing>` consumindo `<PostCard>`
   - Schema JSON-LD `Blog` com `parentOrganization` BeneficioRH (FR54)
   - Metadata: title `Blog | Plano Amil Empresas`, canonical, OG

3. **Página post detail** `app/(marketing)/blog/[slug]/page.tsx`:
   - `generateStaticParams` para todos os posts publicados
   - Render Portable Text com componentes ricos (suportar `<PriceTable>`, `<FAQAccordion>` quando embedded)
   - Hero: title + author byline + datePublished + reading time
   - TOC sticky lateral gerado de h2/h3 do body
   - Schema JSON-LD `BlogPosting` (alias Article): `headline`, `author` (Story 6.9 ref), `datePublished`, `dateModified`, `image` (ogImage), `publisher` (Organization BeneficioRH), `mainEntityOfPage`, `articleSection` (categoria)
   - `BreadcrumbList` schema
   - 404 se post não publicado ou slug inexistente

4. **Página categoria** `app/(marketing)/blog/categoria/[categoria]/page.tsx`:
   - Listing posts da categoria, paginado
   - Schema JSON-LD `CollectionPage` com `parentOrganization` BeneficioRH
   - `generateStaticParams` para 8 categorias do roadmap

5. **RSS feed** `app/(marketing)/blog/feed.xml/route.ts`:
   - RSS 2.0 válido (W3C feed validator)
   - Include últimos 30 posts: title, link, description (excerpt), pubDate, author, category
   - Header `Content-Type: application/rss+xml; charset=utf-8`

6. **Componentes blog** em `components/blog/`:
   - `PostCard.tsx` — card listing (cover + title + excerpt + author + date)
   - `BlogListing.tsx` — grid + paginação
   - `CategoryBadge.tsx` — badge categoria reutilizável

7. **Roadmap editorial** `docs/editorial/blog-roadmap.md`:
   - 30 títulos planejados distribuídos em 8 categorias (alinhar com FAQ Story 4.7):
     1. Carências
     2. Coparticipação
     3. Reembolso
     4. Rede credenciada
     5. Adesão MEI/PME
     6. ANS / Regulamentação
     7. Cobertura
     8. Cancelamento / Portabilidade
   - Cada título com: keyword target, intent (informational/comparative/transactional), Wave alvo (1/2/3), categoria
   - Distribuição alinhada com waves: ~10 títulos Wave 1 cornerstones técnicos, ~10 Wave 2 comparativos, ~10 Wave 3 long-tail

8. **Schema validation CI**: `scripts/validate-schema.mjs` (ou existente) extendido para validar `BlogPosting`, `Blog`, `CollectionPage` em rotas blog — fail CI se algum schema malformado.

9. **Lighthouse**: rota sample `/blog/[slug-mock]` ≥92 em Performance/SEO/A11y/Best-Practices.
