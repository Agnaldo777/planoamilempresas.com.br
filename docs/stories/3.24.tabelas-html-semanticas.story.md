## Story

**As a** decisor PJ acessando tabela de preços via screen reader, leitor de tabela em SERP Featured Snippet, e **@dev (Dex)**,
**I want** todas as tabelas de preços (master, per-estado, per-produto) refatoradas para usar `<table>` HTML semântico nativo com `<thead>`/`<tbody>`/`<th scope="col"|"row">` e `<caption>` descritivo, eliminando div/CSS Grid puro,
**so that** o satélite cumpra FR52 do PRD v1.3.1, ganhe Featured Snippet alvo "plano amil tabela", melhore acessibilidade WCAG 2.1 AA (NFR9) e diferencie de `amilsaudebr.com.br` que usa div/CSS Grid sem semântica.

---

## Strategic Context / Why

Featured Snippet do Google para query "plano amil tabela" e "tabela amil 2026" prefere **`<table>` HTML nativo** — markup div/Grid não é parseado como tabela pelo crawler. `amilsaudebr.com.br` (research `09-amilsaudebr-kitcorretor-deep-scrape.md`) usa div/CSS Grid: gap explorável.

Acessibilidade (NFR9) também depende de markup semântico — screen readers anunciam "tabela 6 colunas 10 linhas" e permitem navegação cell-by-cell apenas em `<table>` real. Story 6.1 (PriceTable) provavelmente usa div/Grid (responsivo mais fácil). Refactor + responsivo via CSS (não markup) é a forma correta.

Schema `Table` é mapeável para rich results — vincula com Story 3.23 (AggregateOffer) e Story 6.1 (Product schema).

## Acceptance Criteria

1. **Refactor `<PriceTable />`** (Story 6.1) `src/components/pricing/PriceTable.tsx`:
   - Markup raiz: `<table>` com `<caption>`, `<thead>`, `<tbody>`
   - `<thead>` contém `<tr>` com `<th scope="col">` por coluna (faixa etária, produto, etc.)
   - `<tbody>` contém `<tr>` com `<th scope="row">` para identificador da linha + `<td>` para valores
   - `<caption>` descritivo: "Tabela Amil PME — São Paulo — Atualizada em [mês/ano]"
   - Mantém props existentes (filtros, dataset Story 6.1)

2. **Responsividade via CSS, não markup:**
   - Desktop: tabela tradicional zebra striping
   - Mobile (≤768px): tabela transformada visualmente em cards via CSS (`display: block` em `<tbody>`, etc.) MAS markup permanece `<table>`
   - Tested em axe-core e manual screen reader (NVDA + VoiceOver)

3. **Outras tabelas** refatoradas:
   - `<RedeCredenciadaTable />` (Story 7.X) — listagem prestadores
   - `<ChangelogTable />` (Story 6.2) — histórico tabela preços
   - `<CarenciaTable />` (Story 3.X cornerstone) — tabela de carências por procedimento
   - `<ComparadorTable />` (Story 3.21) — comparativo lado-a-lado (caso usar table; pode usar grid se Uma decidir cards mobile)

4. **Schema `Table` JSON-LD** (opcional mas recomendado):
   - Wrap parent schema (Product / WebPage) com `mainEntity: { @type: "Table", about: "Tabela de preços Amil PME São Paulo 2026" }`
   - Validar Rich Results

5. **A11y enforcement:**
   - axe-core CI passa em 100% das páginas com tabela
   - Lint rule custom `eslint-plugin-jsx-a11y/no-noninteractive-element-to-interactive-role` cobre + custom rule `eslint-plugin-local/require-table-semantics` para detectar div com `role="table"` (anti-pattern)
   - Manual screen reader test documentado em `docs/qa/accessibility-checklist.md` para cada tipo de tabela

6. **Tests unitários** `PriceTable.test.tsx`:
   - Renderiza `<table>` real (não div)
   - `<th scope="col">` em headers
   - `<caption>` presente e descritivo
   - Filtro mantém estrutura semântica
   - Mobile breakpoint não muda markup (CSS only)

7. **Tests RTL accessibility** `tests/a11y/PriceTable.a11y.test.tsx`:
   - axe-core passa
   - Screen reader output (mock) anuncia tabela corretamente
   - Navegação Tab + Arrow keys funcional (ou documentado fallback)

8. **Featured Snippet validation:**
   - `<table>` markup é detectável via Google Search Console / Rich Results Test
   - Story DoD: screenshot da Tabela Amil de SP em Search Console "table detected" ou Rich Results Test

9. **Performance:**
   - Lighthouse a11y ≥95 (target alta)
   - Lighthouse SEO ≥95
   - LCP <2.0s preservado (tabela é text/HTML, sem imagens)

10. **Documentação** `docs/components/table-best-practices.md`:
    - Markup template `<table><caption><thead><tbody>`
    - Responsivo via CSS guideline
    - Quando usar `<th scope="row">` vs `<th scope="col">`
    - Anti-padrões a evitar (div com role="table")

11. **Migração de PriceTable existente:**
    - Backward compat: testes E2E existentes (Story 6.1 / 6.2) continuam passando
    - Nenhuma regressão funcional (filtros, sort, export PDF)

12. **Fallback gracioso:**
    - Se JS desabilitado, tabela ainda renderiza (RSC server-rendered HTML)
    - Filtros via query param URL (já em Story 6.1) continuam funcionais

13. **NFR9 WCAG 2.1 AA enforcement:**
    - Contraste cell text vs background ≥4.5:1
    - Foco visível em cells navegáveis
    - Headers anunciados corretamente

14. **Compliance schema validation CI** estende detecção `<table>` em pages onde esperamos.

15. **Mobile UX nota:**
    - Mobile transformation é CSS only — `<table>` permanece. Decisão UX (Uma): CSS pattern recomendado é `display: block` no `<tbody>`/`<tr>` + content via `data-label` em cada `<td>` mostrado via `::before`. Documentado em `table-best-practices.md`.
