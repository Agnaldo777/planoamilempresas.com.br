## Story

**As a** decisor PJ buscando rede credenciada específica de produto Amil ("Hospitais D'or no plano Amil", "Amil One rede selecionada") e **@dev (Dex)**,
**I want** páginas dedicadas a sub-redes credenciadas Amil em rotas próprias `/rede/hospitais-dor/`, `/rede/amil-one-rede-selecionada/`, `/rede/clinicas-amil/`, `/rede/amil-facil-rede/`, `/rede/classica/` (ou slugs equivalentes ADR-006), cada uma com schema HealthInsurancePlan + ItemList, conteúdo editorial ≥600 palavras e listagem dataset-driven dos prestadores específicos da sub-rede,
**so that** o satélite cubra FR44, capture gap parcial dos concorrentes (`amilplanos` cobre só `/rede-de-hospitais-dor-amil/`; `planodesaudeamil` mistura sub-redes com tiers) e domine queries de sub-rede específica.

---

## Strategic Context / Why

Sub-redes Amil ("Hospitais D'or", "Amil One Rede Selecionada", "Clínicas Amil Fácil", "Linha Clássica") são **diferenciador de produto** comunicado pela Amil oficial mas mal-coberto por satélites concorrentes:

- `amilplanos.com.br`: tem `/rede-de-hospitais-dor-amil/` única (gap em One/Fácil/Clássica)
- `planodesaudeamil.com.br`: mistura sub-redes com tiers — sem páginas dedicadas
- `amilsaudebr.com.br`: zero páginas de sub-rede

Cada sub-rede tem semântica própria + público específico (D'or = qualidade premium hospitalar; One = rede selecionada premium; Fácil = rede limitada PME barato; Clássica = rede ampla nacional). Páginas dedicadas com **lista real de prestadores filtrada por sub-rede via dataset** (Story 7.1 helpers) + schema rich = moat de 5-7 URLs estratégicas com alto intent comercial.

Story complementa Epic 7 que cobre **rede credenciada agregada por geografia**; esta cobre **rede agregada por produto/sub-marca**.

## Acceptance Criteria

1. **Inventário sub-redes Amil cobertas** (validação Story 7.0a + ADR-006):
   - **Hospitais D'or** (rede própria UnitedHealth — premium hospitalar)
   - **Amil One Rede Selecionada** (premium executivo)
   - **Amil Fácil / Clínicas Amil** (PME budget regional)
   - **Linha Clássica** (rede ampla nacional)
   - **Linha Selecionada** (S380, S450, S750 — rede regional específica)
   - Mínimo 5 páginas; máximo 8 dependendo de validação ADR-006 + advogado co-sign

2. **Rota Next.js App Router** `app/(marketing)/rede/[sub-rede]/page.tsx` (SSG via `generateStaticParams`):
   - Slugs alinhados com ADR-006 (URL-as-trademark policy + 5 mitigações)
   - Slug exemplo: `/rede/hospitais-dor/`, `/rede/amil-one-selecionada/`, `/rede/clinicas-amil-facil/`, `/rede/linha-classica/`, `/rede/linha-selecionada/`
   - Cada página é canônica de si mesma (no canonical externo)

3. **Conteúdo editorial ≥600 palavras únicas por página** (alinhado NFR25):
   - Hero: H1 "Plano Amil [Sub-Rede] 2026 — Rede Credenciada e Cobertura"
   - Seção 1: O que é a sub-rede (origem, posicionamento, público-alvo) — 200 palavras
   - Seção 2: Hospitais/clínicas/laboratórios incluídos — lista dataset-driven via Story 7.1 helpers
   - Seção 3: Diferenças vs outras sub-redes Amil — tabela comparativa breve
   - Seção 4: Produtos Amil que dão acesso a essa sub-rede (mapeamento sub-rede ↔ Bronze/Prata/Ouro/Platinum/Black/etc.)
   - Seção 5: FAQs (4 Q&A) com `<FAQAccordion />` (Story 4.7)
   - Seção 6: CTA cotação WhatsApp + formulário

4. **Listagem dinâmica de prestadores via Story 7.1 helpers:**
   - Usa `getPrestadoresPorRede(rede: RedeAmilNome)` para popular ItemList
   - Top-20 prestadores em destaque + paginação (50/página) ou link "ver todos os N"
   - Cada item linka para `/rede/[uf]/[municipio]/[prestador-slug]/` (Story 7.4)
   - Filtro por UF e tipo (Hospital/Lab/Clínica) usando Story 7.1 helpers `getPrestadoresPorTipo`

5. **Schema JSON-LD rich:**
   - `HealthInsurancePlan` com `name: "Amil [Sub-Rede]"`, `provider: Organization BeneficioRH` (FR54), `network: ItemList` com prestadores
   - `ItemList` com `MedicalOrganization` items (top-20)
   - `BreadcrumbList`: Home → Rede Credenciada → [Sub-Rede]
   - `FAQPage` (4 Q&A)
   - Validar Rich Results Test screenshot ≥3 sub-redes amostradas

6. **Mapeamento sub-rede ↔ produtos Amil:**
   - Constante `src/data/operadoras/amil/sub-redes-to-produtos.ts` mapeia `subRede → produtos[]`
   - Ex: `'hospitais-dor' → ['Amil Black', 'Amil One S6500', 'Platinum Mais']`
   - Validação cruzada com `data/tabelas-amil.ts` em CI (sem produto fantasma)

7. **Internal linking:**
   - Hub `/rede-credenciada/` (Story 7.2) linka às sub-redes em seção dedicada "Por sub-rede"
   - Cada sub-rede linka recíproco para hub
   - Páginas-prestador (Story 7.4) que pertencem a sub-rede linkam para a sub-rede correspondente
   - Cornerstones de produto (ex: `/produtos/amil-black/`) linkam para sub-redes acessíveis

8. **Compliance ADR-006 (URL-as-Trademark policy):**
   - Slugs evitam usar marca "Amil" sozinha como diferenciador único — sempre qualificada (ex: `/rede/amil-one-selecionada/` ok; `/amil/` não ok)
   - 5 mitigações ADR-006 aplicadas: disclaimer, no-logo, no-claim-de-autoridade-oficial, link "ver versão oficial Amil", footer cross-domain `rel="me"`
   - Advogado revisor (Story 2.4) co-sign dos 5 slugs antes de deploy

9. **Disclaimer obrigatório no rodapé da página:**
   - "Sub-rede sujeita a alterações pela Amil. Confirmar disponibilidade no app oficial Amil antes de uso. Última atualização: [data]."
   - Link discreto "Ver lista oficial Amil"
   - Componente `<Disclaimer type="rede-sub-marca" />` reusado

10. **NFR25 anti-cookie-cutter:**
    - Conteúdo editorial **escrito human por sub-rede** (não template repetido) — 600 palavras únicas, refletindo posicionamento real de cada sub-rede
    - Story 5.7 audit incluirá amostragem destas 5-8 páginas

11. **Performance:**
    - Lighthouse ≥92 (alinha NFR1)
    - LCP <2.0s
    - Imagens via `next/image` (logos sub-rede stylized — sem logotipo Amil oficial conforme NFR8)

12. **Tests E2E** Playwright:
    - 5 sub-redes navegáveis
    - ItemList renderiza N prestadores corretos por dataset
    - Links internos funcionais (sample 3)
    - Schema válido em network response

13. **Tests unitários:**
    - Mapeamento sub-rede→produtos
    - `getPrestadoresPorRede('AMIL S750 QP')` retorna count esperado
    - Slug normalization

14. **Validação dataset coverage:**
    - Cada sub-rede listada precisa ter ≥30 prestadores no dataset (mínimo viável). Sub-redes com <30 são noindex + canonical para `/rede-credenciada/`

15. **Lint rule custom** — links externos para amil.com.br devem ter `rel="nofollow noopener"` quando referenciados como "site oficial".
