## Story

**As a** stakeholder,
**I want** acessar uma página canary "em breve" no domínio `planoamilempresas.com.br` deployado e um endpoint `/api/healthz` retornando JSON válido,
**so that** eu valide visualmente que o fork limpo + pipeline CI/CD + DNS + Vercel estão operacionais antes de investir tempo em conteúdo Amil.

---

## Strategic Context / Why

Esta story é o **primeiro deploy funcional pós-fork** — encerra o Epic 1 fundacional do ponto de vista visual. Sem canary visível, o stakeholder não tem confirmação tangível de que a infraestrutura existe. Com canary + healthz endpoint, todas as Stories 1.5+ (analytics, LGPD, footer, SEO foundation) têm página de origem para hookar. É também o gate final do "Bradesco strip" — qualquer string Bradesco que sobreviveu ao Story 1.1 será apanhada nesta story via grep audit ao build de produção.

Lighthouse target ≥95 nesta página simples é facilmente atingível e estabelece baseline de performance que deve ser preservado em todas as stories subsequentes.

---

## Acceptance Criteria

(Fonte: PRD v1.2.2 Epic 1 Story 1.4, 10 ACs)

1. Rota `/` (page server-rendered via React Server Component) com:
   - Título "Em breve — planoamilempresas.com.br"
   - Tagline curta editorial
   - Data da última build via `process.env.VERCEL_GIT_COMMIT_SHA` (curto) + `Date.now()` injetado no build
2. Rota `/api/healthz` (Next.js Route Handler com Edge Runtime) retorna JSON válido:
   ```json
   { "status": "ok", "version": "<git-sha>", "timestamp": "<ISO8601>", "environment": "production|preview|development" }
   ```
   - Verificável via `curl https://planoamilempresas.com.br/api/healthz` retornando 200
3. Página 404 custom (`app/not-found.tsx` herdada do clone, limpa de Bradesco) com link para `/`
4. Página de erro (`app/error.tsx`) com logging Sentry condicional (se `SENTRY_DSN` env existir) — Story 1.5 adicionará valor real; aqui apenas a integração condicional
5. Meta tags básicas no `layout.tsx`:
   - `<title>` único
   - `<meta name="description">` < 160 chars
   - `<meta name="viewport">`, charset
   - `og:title`, `og:description`, `og:image` (placeholder), `og:type=website`, `og:url`
6. Favicon placeholder (pode ser genérico, ex: ícone simples gerado ou padrão Next.js)
7. HTTPS ativo com cadeado no browser (validação visual + `curl -I`)
8. Lighthouse Performance ≥95 nesta página simples (target facilmente atingível em página trivial)
9. Core Web Vitals: LCP < 1s, CLS = 0, INP < 100ms (Lighthouse + WebPageTest validation)
10. **Grep de validação confirma:**
    - Zero strings `Bradesco` em build artifact (`.next/`)
    - Zero referências a `amil-saude` no código atual
    - Zero IDs Bradesco em analytics (env vars são placeholder, valores reais virão Story 1.5)

### Key Technical Context

**Edge Runtime no `/api/healthz`:** otimização de latência (TTFB <50ms global) + zero cold start. Health check é caso de uso ideal para Edge.

**Build info exposure:** `VERCEL_GIT_COMMIT_SHA` é seguro de expor publicamente (revela commit SHA mas repo pode ser privado). Não expor `VERCEL_GIT_COMMIT_AUTHOR_NAME` ou similar.

**Sentry condicional:** evita Sentry quebrar build se `SENTRY_DSN` ainda não foi configurado em Story 1.5.
