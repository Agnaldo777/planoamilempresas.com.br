## Story

**As a** SEO programmatic strategist,
**I want** ativar `BUILD_FULL_PROVIDERS=true` + `PHASE_2_ENABLED=true` em CI, gerar 9.325 prestadores SSG com schemas markup completos por subtipo, validar build time ≤25min via bundle analyzer, rodar Lighthouse em 10 URLs amostrais estratificadas e popular sitemap shard `app/sitemap-prestadores.xml` com `<lastmod>` real,
**so that** o satélite capture o long-tail prestador-by-name (~9.325 search queries únicas) com schemas rich snippet validados e sem regressão de performance, completando o Ataque 2 do Epic 7.

---

## Strategic Context / Why

Esta story isola o **risco alto / valor alto** do split de 7.4 — depende de:

1. Deploy Cloudflare Workers verde (Stories 1.2 + 1.3 v2) — Hobby tier Vercel substituído por Workers (ADR-011)
2. Decisão go/no-go Phase 1+2 simultâneo após primeira Phase 1 SERP measurement

## Acceptance Criteria

Esta story herda os seguintes AC da master 7.4 (escopo: pipeline full pós-deploy):

3. **(AC3 master) Schema markup por tipo inferido** via `<PrestadorJsonLd />` discriminated union (6 branches Schema.org canonical: Hospital, MedicalClinic, MedicalLaboratory, EmergencyService, MedicalImagingService, MedicalOrganization fallback) + branch Hospital com prop opcional `medicalSpecialty: { name: 'Obstetrics' }` para tipo `'Maternidade'`.

3-bis. **(AC3-bis master) `<OrganizationJsonLd />` reuso transversal** — render em todas as 9.325 páginas-prestador com props default (mitigação 2 ADR-006).

4. **(AC4 master) `generateMetadata({params})`** produz title/description/canonical específicos:
   - Title: `{Nome Prestador} em {Município}/{UF} — Rede Credenciada Amil`
   - Description (~155 chars) com bairro + tipo + redes top-3
   - `canonical: /rede/[uf]/[municipio]/[prestadorSlug]/`
   - openGraph + twitter populados

6. **(AC6 master) NoIndex anti-thin** — município com <`MIN_PRESTADORES_MUNICIPIO_PARA_PRESTADOR_INDIVIDUAL=5` prestadores → `noindex,follow` + canonical para cidade-pai (Story 7.5). Constante centralizada em `src/config/seo.ts`.

7. **(AC7 master) Integração templates 7.3** — switch por `tipoInferido` renderiza `<HospitalContent />` / `<LaboratorioContent />` / `<ClinicaContent />` / `<OutroContent />` com ≥600 palavras únicas + ≥8 interpolações nome+bairro+UF.

10. **(AC10 master) Build time targets:**
    - Phase 1 build ≤**25 minutos** em CI Cloudflare Workers (mensurado via `time pnpm build`)
    - Phase 2 build separado ≤15 minutos
    - Falha gate se Phase 1 >30min

11. **(AC11 master) Performance gates** — 10 URLs amostrais estratificadas (top-5 cidades + 5 tail) em Lighthouse CI: Perf ≥92, LCP ≤2.0s, INP ≤200ms, CLS ≤0.05.

13. **(AC13 master) Mapa aproximado Leaflet/OSM** com centroide do bairro/município + aviso "localização aproximada" + lazy-load (sem bloquear LCP).

14. **(AC14 master) CI gate full** — PR não merge sem (a) build verde Phase 1 + Phase 2, (b) lint:strict, (c) tests verdes, (d) Lighthouse 10 URLs, (e) Schema.org validator em 5 prestadores estratificados, (f) noindex test fixture passando.

### AC adicional desta story

B3. **Schema.org Rich Results screenshot** anexado em PR comprovando validação ≥5 URLs estratificadas (1 por subtipo principal).
